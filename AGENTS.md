# Montran Social Design Workflow

This folder documents how agents should create Montran social media designs in Figma from Notion content.

## Primary Sources

- Figma file: https://www.figma.com/design/9pzw3P4vngkkAiitYtcgwn/Montran-Social-Media-Templates
- Working page pattern: monthly pages such as `Apr 2026`, `Mar 2026`, and `Feb 2026`
- Template reference page: `Social Templates`
- Content source: Notion content pages shared by the user
- Generic image library: `/Users/oceancheung/Library/CloudStorage/OneDrive-SharedLibraries-MontranCorporation/Marketing & Communications - Documents/Assets Library/Images/_Generic`

## Core Rules

- Use `Social Templates` as the design reference library, not as the final destination.
- Create actual post layouts on the relevant monthly page.
- Place content by planned content slot/date in Figma, even if the actual posting date later changes.
- Preserve row order: `EN -> EN slides -> ES -> ES slides -> FR -> FR slides -> Web`.
- Build English first unless the user explicitly asks otherwise.
- Duplicate the approved English layout for Spanish and French.
- Use `Cover` only for the opening post cover.
- Add `Web` square assets only for thought leadership or web-published pieces.
- Ignore `Awards` unless explicitly requested.

## Content Mapping

- `Headline` usually drives the cover slide.
- `Content` drives the carousel body slides.
- `Caption` is publishing copy and should not be placed in the design unless requested.
- `Concept` is creative direction and should influence layout, imagery, and emphasis.
- Languages in Notion indicate which versions are needed, but translated copy may need to be produced by the agent.

## Typography Rules

- Use soft returns, equivalent to Shift+Enter, for manual line breaks inside text blocks.
- Do not use paragraph returns where a line break is meant to preserve template spacing.
- If translated text is too long, first shorten the copy while preserving the message.
- If it still does not fit, switch to an available Montran/Figma library text style size.
- Do not leave arbitrary custom font sizes when a library style is available.

## Required Checks

- Review the existing Figma layout before creating or editing slides.
- Browse previous monthly designs when choosing layouts, not only `Social Templates`.
- Check each new or translated slide visually after editing.
- Confirm there are no extra dots, lines, duplicated decorative elements, or misplaced images.
- Confirm row order after adding language versions.

See the files in `docs/` for detailed workflow, design, translation, asset, and QA guidance.
