# QA Checklist

Run this checklist before handing a Figma design back to the user.

## Content

- The design uses the correct Notion page.
- The design is placed on the correct monthly page and planned date row.
- The cover reflects the headline or approved distilled headline.
- Body slides follow the Notion content structure.
- Caption copy is not accidentally placed into the design.
- Technical terms and acronyms are preserved.

## Layout

- The row order is correct: `EN -> EN slides -> ES -> ES slides -> FR -> FR slides -> Web`.
- Auto-layout child order matches the visual order.
- No existing unrelated content was overwritten.
- Slides use appropriate templates based on content role.
- The carousel has visual pacing and does not repeat the same layout unnecessarily.

## Text

- Manual line breaks use soft returns.
- Text is not clipped or overflowing.
- Spanish and French translations are concise enough to fit.
- Library/published text styles are used when a smaller size is needed.
- No arbitrary custom font sizes are left behind when a library style is available.
- Accents, punctuation, and special characters render correctly.

## Visuals

- Images are present only on intended slides.
- Image crop, mask, and overlay are correct.
- No extra template elements remain, such as stray dots, lines, or hidden duplicates.
- Decorative elements align with the template.
- Final slide feels like a clear takeaway, proof point, or CTA.

## Review

- Screenshot the full row after edits.
- Screenshot any slide with dense text or replaced imagery.
- Confirm English first, then language versions.
- If only English was requested, preserve ES, FR, and Web placeholders.
- Mention any unresolved assumptions in the final handoff.
