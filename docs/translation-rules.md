# Translation Rules

## Workflow

Build and approve English first. Then duplicate the English design for Spanish and French so the language versions preserve layout, pacing, and visual hierarchy.

Use this order in the Figma row:

`EN -> English slides -> ES -> Spanish slides -> FR -> French slides -> Web`

## Translation Approach

Translate the approved English slide copy, not raw Notion content. The English design may already be distilled from the source article, and the translations should match that designed narrative.

Priorities:

- Preserve meaning.
- Preserve hierarchy.
- Keep copy concise enough to fit.
- Keep technical acronyms unchanged unless there is a standard localized usage.
- Avoid overly literal translations that sound unnatural.

## Fitting Text

If Spanish or French is too long:

1. Condense the sentence while preserving the point.
2. Use fewer modifiers and avoid repeated context.
3. Keep line breaks intentional and use soft returns.
4. If copy still cannot fit, use an available Montran/Figma library text style size.
5. Do not leave arbitrary custom font sizes when a linked library style is available.

## Typography

Use soft returns, equivalent to Shift+Enter, for manual line breaks. Do not use paragraph returns for visual line breaks, because they create incorrect spacing inside the template.

When text requires a smaller size, prefer the available published text styles used by the template or Montran Assets Library styles. In the Apr 27 carousel, examples of published display styles used by the template included:

- `Special Headings/Display 5`
- `Special Headings/Display 6`
- `Special Headings/Display 7`

## Quality Check

After translating, visually check each slide. Look for:

- clipped accents or descenders
- overly dense text blocks
- bad line breaks
- titles wrapping awkwardly
- inconsistent font size relative to the English slide
- text shifted out of alignment by auto-layout changes
