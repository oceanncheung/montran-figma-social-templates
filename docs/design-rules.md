# Design Rules

## Layout Selection

Choose layouts based on the content structure, not only on available template names.

- Use `Cover` frames only for opening slides.
- Use `Page Section` and section layouts for body content when the Notion structure has multiple sections.
- Use stat-heavy templates for numbers, market size, growth, payment values, or adoption proof.
- Use progression/path templates for staged modernization, roadmaps, migration paths, or maturity curves.
- Use proof/CTA templates for final slides with examples, market validation, or key takeaways.

## Visual Pacing

A good carousel should not feel like repeated text blocks. Vary the slide roles:

- Cover: strong headline and visual hook.
- Problem/tension: explain why the topic matters now.
- Data/proof: use numbers or concrete evidence.
- Framework/path: show how the audience should think about the topic.
- Closing: implication, action, or memorable takeaway.

## Previous Designs

When a template feels too dry, inspect prior monthly pages and reuse successful patterns. This is preferred over forcing all posts into the generic `Social Templates` layouts.

Known examples from this workflow:

- Apr 14 infrastructure carousel: useful for large-stat and implication layouts.
- Apr 27 connector-to-GPH carousel: useful for editorial cover, tension slide, scale slide, staged path, and proof/CTA close.

## Imagery

Images should support the story. Use banking, infrastructure, payments, business, or regional context where appropriate. Avoid generic images that do not add meaning.

Only replace images on slides that are intended to use images. For the Apr 27 workflow, the user clarified that imagery should be on page 1 and page 3 only.

## Decorative Elements

Do not leave stray decorative elements from copied templates. Check especially for:

- extra orange or teal dots
- duplicated lines
- hidden/unmasked image leftovers
- decorative objects copied from a source slide but not intended for the new slide
- elements that fall outside the slide mask or frame
